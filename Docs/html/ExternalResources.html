External resources

This section contains links to external resources documenting the hardware of the AR Drone, software implementations made by paparrazi and general quadcopter information.

General resources:

Parrot AR Drone 2.0 official specifications

The parrot website contains the specifications of the AR Drone 2.0 with details about the camera, motors, processing boards and construction

IFIXIT AR.Drone 2.0 Repair Guide

IFIXIT provides a full guide on how to repair the AR Drone 2.0. From replacing a single propellor to the entire navigation board.

TI Processor resources

TI AM35x-OMAP35x-PSP 03.00.00.05 UserGuide

The AR Drone 2.0 uses the TI OMAP 35 as it's main processing unit. The user guide includes links to video capture and resizing drivers

TI User guide for the TMS320C64x/C64x+ DSP

The AR Drone 2.0 uses the C64x digital signal processing unit.

TI Programmers guide for the TMS320C6000

The AR Drone 2.0 uses the C64x digital signal processing unit which is based on the TMS320C6000

Sensor data sheets

Bosch BMA150 Digitial, triaxial accelration sensor Data sheet

The bosch BMA150 is the three axis accelerometer used in the IMU board of the AR Drone 2.0

Invensense IDG500 Dual-Axis Gyro Product Specification

The Invensense IDG500 is a 2 axis gyroscope, used to measure the velocity of the drone in the X and Y plane.

Epson XV-3700 Gyro Sensor Data sheet

The Epson XV-3700 is a 1D gyroscope which is used to measure the velocity in the Z plane.

Video resources

YCbCr Color space

The AR Drone 2.0 captures video in the YCbCr (also known as YUV) colorspace. This means that for a number of applications the color space has to be converted to RGB. This Wikipedia link provides background on the YCbCr colorspace and conversions to other color spaces.

4:2:2 subsampling explanation

The front camera of the AR Drone 2.0 captures video at a resolution of 1280 * 720 pixels. The horizontal resolution however is interlaced meaning that only the chroma value covers the full 1280 pixel width. Wikipedia provides insight on how this method of subsampling works.

TI Display driver guide using Video4Linux

The TI OMAP processor within the AR Drone 2.0 has built in support for video streaming using the Video 4 Linux 2 API. TI provides a guide on how to use it.

Video4Linux2 Documentation

The Video 4 Linux 2 documentation contains a lot of information on what function calls to use to acces the camera output.
